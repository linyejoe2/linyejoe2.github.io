### 虛擬站台(VirtualHost)
**cd /etc/apache2/site-available**這就是虛擬站台放置的位置
**000-default.conf**是虛擬站台設定檔的範本

最基本的配置
**ServerName**是你虛擬站台的名稱，就是一台虛擬電腦的電腦名稱一樣，等等DNS解析完後就可以直接打主機名稱連到虛擬站台了
**DocumantRoot**是你index.html放置的位置(注意不能跟別的虛擬站台重複

打完wq儲存
然後用**a2ensite 000-default.conf**登入你的設定檔
**a2dissite**可以關閉你的虛擬站台
記得**service apache2 restart**重新啟動服務

### 使用者登入
有些機密網站我們會希望有使用者登入的功能
你需要兩份文件，第一份是.htpasswd，裡面放的是有權限登入的使用者，第二份是.htaccess，裡面放的是你想要限制登入的網站的設定
:::info
.htpasswd
**htpasswd -c /etc/apache2/.htpasswd user1**
-c代表新增文件 如果還要加入新使用者就不用加-c了
:::
:::success
.htaccess 
把.htaccess放在/var/www/html(或是你自訂的放置位置)
```cmake=
AuthType Basic
AuthName "Login"
AuthUserFile /etc/apache2/.htpasswd
Require valid-user
```
:::

## DNS(bind9)
**apt-get install bind9**下載bind9
**cd /etc/bind/**進入bind的資料夾裡
**named.conf**是你的主設定檔
新增一個ZONE的範例
```cmake=
zone "tnsc.tw" {
        type master;
        file "/etc/bind/db.tnsc.tw";
};
```
wq後**vim db.tnsc.tw**就是你的正解資料庫檔案
db.local是你的範例，複製後開啟
```sas=
$TTL        604800
@           IN     SOA    tnsc.tw. linyejoe.gmail.com.(
                   2
              604800
               86400
             2419200
              604800  )
@           IN     NS     tnsc.tw.
@           IN     A      192.168.1.87
www.tnsc.tw. IN    A      192.168.1.87
web.tnsc.tw. IN    A      192.168.1.87 
```
:::info
常見正解檔相關資訊
+ SOA 管理網域的七個參數
    1 Master DNS伺服器
    2 email
    3 序號(判斷檔案新舊，越大越新
    4 Refresh(更新)
    5 Retry(重新嘗試連接Master)
    6 Expire(失效)
    7 TTL
+ A  ipv4的ip位址
+ NS 管理網域的伺服器主機名稱

+ MX 郵件伺服器主機名稱
+ CNAME 主機別名 (不常用)
:::
wq儲存，這樣就建立了三筆分別為
+ tnsc.tw
+ www.tnsc.tw
+ web.tnsc.tw
的正解，都導向192.168.1.87，配合剛剛的虛擬主機就可以設定成不同網站了

### view 
view可以根據主機位址分別給予不同的解析

```cmake=
view "lan" \\名字
{
    match-clients \\在這個view裡的網段
    {
        192.168.100.0/24; 
        127.0.0.0/24;
    };

    zone "." 根解析伺服器，兩邊都要有
    {
        type hint;
        file "/etc/bind/db.root";
    };
    
    zone "skills39.org"
    {
        type master;
        file "/etc/bind/db.skills39.org.lan";
    };
};

view "wan" 
{
    match-clients 
    {
        any; \\所有網段
        ! 192.168.100.0/24; \\!=反向
        ! 127.0.0.0/24;
    };

    zone "." 
    {
        type master;
        file "/etc/bind/db.root";
    };
    
    zone "skills39.org"
    {
        type master;
        file "/etc/bind/db.skills39.org.wan";
    };
};
```

### iptables儲存

### iptables參數表
:::info
-t<表>：指定要操縱的表；
-A：向規則鏈中添加條目；
-D：從規則鏈中刪除條目；
-I：向規則鏈中插入條目；
-R：替換規則鏈中的條目；
-N：創建新的用戶自定義規則鏈；
-L：顯示規則鏈中已有的條目；
-v：列出更多資訊；
-P：定義預設政策；
:::
:::danger
-F：清除規則鏈中已有的條目；
-Z：清空規則鏈中的數據包計算器和字節計數器；
-X ：殺掉所有使用者自訂的表；
:::
:::warning
-p：協議類型；
-s：來源ip地址，可指定IP或是網域；
-d：目標ip地址，可指定IP或是網域；
--sport 埠口範圍：限制來源的埠口號碼，EX:1024:65535；
--dport 埠口範圍：限制目標的埠口號碼；
-i<INPUT>：指定數據包進入本機的網絡接口；
-o<OUTPUT>：指定數據包要離開本機所使用的網絡接口；
-j：指定動作(放最後)；
:::
:::success
-m ：一些 iptables 的外掛模組，主要常見的有：
     state ：狀態模組
     mac   ：網路卡硬體位址 (hardware address)
     
--state ：一些封包的狀態，主要有：

     INVALID    ：無效的封包，例如資料破損的封包狀態
     ESTABLISHED：已經連線成功的連線狀態；
     NEW        ：想要新建立連線的封包狀態；
     RELATED    ：表示這個封包是與我們主機發送出去的封包有關
:::

### NAT(網路位址轉換)
要啟用nat 首先你要開啟電腦的封包轉送 就是像把電腦變成一台router一樣 要啟用封包轉送，到/etc/sysctl.conf中修改
**net.ipv4 .ip_forward = 1**儲存後要輸入
**sysctl -P /etc/sysctl.conf**來修改設定

**iptables -t nat -A POSTROUTING -s 192.168.1.0/24 -o ens33 -j MASQUERADE**就可以讓這台電腦後面的192.168.1.0網域透過這台電腦上網

### Filter(過濾封包)
**iptables -A INPUT -j REJECT**禁止所有外部的訪問

**iptables -A INPUT -p tcp --dport 80 -m state --state NEW -i ens33 -s 192.168.1.254 -j ACCEPT**允許外部訪問port80(也就是http)

**iptables -A INPUT -m state --state INVALID -j DROP**丟棄不合法封包

## 網路上的芳鄰(SAMBA)

**apt-get install samba**安裝samba
**cd /etc/samba/**進入主資料夾
**cp smb.conf smbback**增加備份檔
**rm smb.conf**再**vim smb.conf**自己打一個設定檔
```sas=
[global] \\全域設定
    map to guest = bad user \\不用login就可以登入
    security = user \\安全級別
    workgroup = network43.tw \\工作群組   

[public] \\你要分享的資料夾的名字
    path = /share/mkt \\共享資料夾位置
    browseable = yes \\可以被看見 
    readonly = yes \\只能讀
    guest ok = yes \\允許匿名登入 
    write list = administrator \\可以寫的使用者
    valid users = administrator , user1 \\有權限瀏覽的使用者
```
:::info
11,12行功能執行必須先新增使用者

如果你需要新增可以使用森巴的使用者請依照以下的步驟做
1 **adduser user1**建立一個Unix使用者
2 **sampasswd user1**建立一個森巴使用者
3 如果你想要建立一個Windows不用驗證即可查看的森巴資料夾，請務必確保森巴使用者與windows使用者為同帳號同密碼
:::

## 單機模式 

當你因為各種原因無法登入root時使用
重開機之後在grub畫面按e進入參數編輯畫面
在linux開頭的那一行最後加入**single**代表你要進入單機模式
然後f10啟動就可以登入root修改錯誤了!

## 修改ens33 to eth0
greb 的組態存在於 **/boot/grub/grub.cfg** 但該檔案不可以人工方式修改 
所以經由修改 **/etc/default/grub**裡的變數來達成
在GRUB_CMDLINE_LINUX項加上 **net.ifnames=0 biosdevname=0**
修改完後 **grub-mkconfig -o /boot/grub/grub.cfg** 可以讓系統載入你的設定檔

## Root登入圖形介面
修改 **etc/gdm3/daemon.conf**
在[secunity]裡加入**AllowRoot = true**
之後修改 **etc/pam.d/gdm-password**
在 **auth required pam_succeed_if.so user != root quiet_success** 裡加入註解後reboot即可
